# MCP-S Licensing Policy

## Recommended project license

MCP-S uses the Apache License, Version 2.0, for:

- source code;
- reference implementation;
- conformance tests;
- conformance vectors;
- schemas;
- protocol specifications;
- security boundary documents;
- proposal documents;
- examples and demo servers.

Using Apache-2.0 throughout avoids ambiguity between normative specifications, executable conformance material, and explanatory documentation.

## Why Apache-2.0?

Apache-2.0 is recommended because it is permissive, widely accepted, includes an explicit patent grant, and is suitable for protocol reference implementations and standards-adjacent projects.

## SPDX identifiers

New source and documentation files should include an SPDX identifier.

Rust/code:

```rust
// SPDX-License-Identifier: Apache-2.0
```

Markdown:

```markdown
<!-- SPDX-License-Identifier: Apache-2.0 -->
```

## Contributor licensing

Contributions are accepted under Apache-2.0 unless explicitly stated otherwise in writing.

See `CONTRIBUTING.md`.
